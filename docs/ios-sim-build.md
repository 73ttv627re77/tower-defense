**TL;DR:** The iOS Simulator build does not work for this project on Godot 4.6.3 with stock scons. Use the **macOS export** as the development inner loop (Godot's official recommendation for Apple Silicon Macs), and ship the **iOS device export** to TestFlight / App Store for iOS validation.

## Why the iOS Simulator build is blocked

Per the official Godot 4.6 docs (Compiling for iOS, Compiling for iOS Simulator):

> **Compiling for the iOS simulator is currently not supported as per GH-102149. Apple Silicon Macs can run iOS apps natively, so you can run exported iOS projects directly on an Apple Silicon Mac without needing the iOS simulator.**

The upstream problems compound:

1. **Official 4.6.3 export template's sim slice is broken** â€” `libgodot.ios.release.xcframework/ios-arm64_x86_64-simulator/libgodot.a` is `x86_64` only, not `arm64`. Tracked as [godotengine/godot#118161](https://github.com/godotengine/godot/issues/118161). On Apple Silicon, the simulator picks the arm64 slice and gets nothing â†’ linker dies.

2. **The device export bundles Mac Catalyst shim code** â€” the `tower-defense-ios` binary inside `tower-defense-ios.app` contains both `OS_IOS` / `UIApplicationDelegate` and `OS_MacOS_NSApp` / `NSApplication` / `NSMenuBarPresentationInstance` symbols. On the iOS Simulator (no Catalyst runtime), the Catalyst path is taken, `_RegisterApplication` aborts, SpringBoard surfaces it as a workspace transition failure. See `~/Library/Logs/DiagnosticReports/tower-defense-ios-2026-06-09-090717.ips` for an example live crash report.

3. **Building a custom sim template from source hits a Swift Concurrency link failure** â€” scons does build an arm64 sim slice successfully (`scons platform=ios target=template_release ios_simulator=yes`), and PR #102179 (in 4.6.3-stable) does disable Metal/Vulkan for sim. But Godot's `drivers/apple_embedded/app.swift` (`godot_swift_module` SwiftUI app) references Swift Concurrency runtime (`_swift_task_isCurrentExecutor`, `MainActor.shared`, `UIApplicationDelegateAdaptor`) that the .xcodeproj generated by the iOS export doesn't link correctly when targeting `iphonesimulator`. Final link error: `Undefined symbols for architecture arm64` in `app.ios.template_release.arm64.simulator.o`.

## What we tried

1. **Diagnose the SBMainWorkspace launch failure.** Real root cause: the device-built iOS .app's engine binary contains Catalyst/macOS code paths that abort on the iOS Simulator. Tracked in [issue #10](https://github.com/73ttv627re77/tower-defense/issues/10).

2. **Build a custom Godot 4.6.3-stable iOS sim export template from source.** Compiled `scons platform=ios target=template_release ios_simulator=yes` successfully (2157 .o files, 160 MB `libgodot.ios.template_release.arm64.simulator.a`). Patched the official 4.6.3 `ios.zip` export template to swap the broken x86_64 sim slice for our arm64 sim .a, and updated the xcframework Info.plist to declare `arm64` only. Re-ran `godot --export-debug iOS`. Patched the regenerated `builds/tower-defense-ios.xcframework` sim slice too. Re-ran `xcodebuild -destination 'generic/platform=iOS Simulator' -configuration Release ARCHS=arm64`. The link fails on the Swift Concurrency symbols mentioned above. The project won't link a sim-targeted .app with the current Godot 4.6.3 sources + Godot-generated .xcodeproj + Apple Silicon Mac.

3. **Run the iOS .app directly on Mac via the iOS-on-Mac runtime.** `open -a` rejects it: `cannot be opened because it has an incorrect executable format` (Mach-O `LC_BUILD_VERSION platform=2`/PLATFORM_IOS, not the platform 1 or 6 the Mac loader accepts).

## Working alternatives

### Recommended: macOS export for the inner loop

`builds/tower-defense-macos.zip` is a 73 MB universal binary (x86_64 + arm64), `Apple M4` compatible, runs Metal Forward+ with the Godot 4.6.3 stable engine. Verified launching today (2026-06-09 09:30 BST):

```
Godot Engine v4.6.3.stable.official.7d41c59c4 - https://godotengine.org
Metal 4.0 - Forward+ - Using Device #0: Apple - Apple M4 (Apple9)
```

The macOS export uses the same GDScript, same sprites, same scene tree as the iOS export. Gameplay iteration on macOS is the official Godot team's recommendation for Apple Silicon Mac development.

### TestFlight / real iPhone for iOS validation

The device .ipa (`builds/tower-defense-ios.ipa`, 32 MB) is properly signed (Apple Development, team `56EXG84N59`, embedded profile) and was designed to run on real hardware. The Catalyst shim is dormant on real iOS â€” `UIApplication` handles everything. Use TestFlight (or direct install via Xcode) to validate iOS-specific behaviour.
